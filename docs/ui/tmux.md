# Tmux

Tmux is the persistent terminal layer.

It keeps terminal work alive across shell reloads, terminal crashes, SSH disconnects, and long-running sessions.

The setup is small:

- `Ctrl-a` prefix
- vi-style copy mode
- Vim-aware pane navigation
- automatic session restore
- generated theme injection

## File

```text
.tmux.conf
```

Theme colors are loaded separately:

```tmux
source-file ~/tmux/theme.conf
```

`theme.conf` is generated by the theme system. The main config owns behavior. The generated file owns colors.

## Core behavior

| Setting      | Value                        |
| ------------ | ---------------------------- |
| shell        | `/bin/zsh`                   |
| prefix       | `Ctrl-a`                     |
| window index | starts at `1`                |
| pane index   | starts at `1`                |
| mouse        | enabled                      |
| clipboard    | enabled                      |
| copy mode    | vi                           |
| terminal     | `xterm-256color` + truecolor |

The config wipes default keybinds first, then applies its own.

## Window switching

Windows start at `1`.

Tmux already supports numbered window switching by default once the prefix is set.

| Bind       | Action             |
| ---------- | ------------------ |
| `Ctrl-a 1` | switch to window 1 |
| `Ctrl-a 2` | switch to window 2 |
| `Ctrl-a 3` | switch to window 3 |
| `Ctrl-a 4` | switch to window 4 |

## Prefix binds

| Bind     | Action                     |
| -------- | -------------------------- |
| `Ctrl-a` | send prefix                |
| `h`      | previous window            |
| `l`      | next window                |
| `n`      | new window in current path |
| `\|`     | horizontal split           |
| `-`      | vertical split             |
| `c`      | split and run `git commit` |
| `b`      | break pane                 |
| `j`      | join pane from prompt      |
| `r`      | reload config              |

New panes inherit the current working directory.

## Pane resizing

Global binds:

| Bind    | Action       |
| ------- | ------------ |
| `Alt-h` | resize left  |
| `Alt-j` | resize down  |
| `Alt-k` | resize up    |
| `Alt-l` | resize right |

## Copy mode

Copy mode uses vi keys.

| Bind    | Action              |
| ------- | ------------------- |
| `v`     | begin selection     |
| `y`     | copy to clipboard   |
| `Enter` | copy to clipboard   |
| `0`     | jump to indentation |
| `Space` | jump again          |

Clipboard integration uses `xclip`.

## Vim-aware navigation

`Ctrl-h/j/k/l` behaves intelligently.

If the active pane is Vim or Neovim, the key is passed into the editor.

Otherwise, Tmux switches panes.

That keeps editor splits and terminal panes on the same muscle memory.

## Plugins

TPM installs itself automatically if missing.

Plugins:

```tmux
tmux-plugins/tpm
tmux-plugins/tmux-sensible
tmux-plugins/tmux-resurrect
tmux-plugins/tmux-continuum
```

Continuum restore is enabled:

```tmux
set -g @continuum-restore 'on'
```

So sessions survive restarts automatically.

## Status bar

The right side is disabled completely.

```tmux
set -g status-right ""
set -g status-right-length 0
```

The shell prompt already carries time, directory, Git state, and runtime context through Starship. Duplicating that inside Tmux adds nothing.

Colors, pane borders, tabs, and messages come from:

```text
~/tmux/theme.conf
```
